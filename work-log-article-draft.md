## 構成

### 導入: きっかけ

KITAcoreさんのnote記事 #300「ダッシュボードを"分析画面"から"ハブ"に作り直した話」を読んだ。
スペシャルサンクスに自分の名前があった。自分のダッシュボードがKITAさんのベースになっていた。
→ 自分のダッシュボードは「分析画面」のまま止まっていた。KITAさんはその先に行っていた。

### 1. 全部見せた

Claude Codeに、おはようカノジョに関わるすべてのプロジェクトを見せた。

- X Bot本体（ohayo-kanojo）
- ダッシュボード（note-stats-tracker）
- ミニログPWA（積み上げ前）
- 積み上げノート（note-follower-pwa）
- おはヨミ（ohayomi）
- おへんじ帖（ohenji-note）
- note-tag-collector
- Kindle書籍「朝7:30の彼女」
- 運営ドキュメント（schedule.md, dashboard-checklist.md, article-categories.md）

コードだけではなく、毎日の運用フロー、週末の作業手順、ミニログの入力項目まで。
なぜそうしたか。**ダッシュボードはツール単体ではなく、運用の中の1ピースだから。**
全体を知らないと「何を見せるべきか」が設計できない。

### 2. 課題が見えた

全体を俯瞰したことで、現状のダッシュボードの問題が明確になった。

- **チェックリストは設計済みなのに、画面がそれに対応していない**
  - 「平日は1分で状態メモをコピーするだけ」と書いてあるのに、全パネルが並んでいる
- **散布図が機能していない**
  - 自己紹介記事が外れ値で、他100本が左下に団子状態
- **wrappedとの中途半端な分離**
  - キャラ演出はwrappedだけ。分析はダッシュボードだけ。どちらも半端
- **ツール間の役割が曖昧**
  - フォロワー推移はダッシュボード？積み上げノート？両方にある？

### 3. ワークフローからタブを設計した

「何をいつ見るか」から逆算してタブを設計した。

最初は Daily / Weekly / Monthly / Deep Dive の4タブ。

**Weekly が膨らみすぎた。** フォロワー推移、スキしてくれた人、記事振り返り、カテゴリバランス、アクション。全部入れたら「コミュニティ活動」と「記事戦略」が混在した。

「スキ返しは毎日やるべきでは？」という問いで、Activityタブが生まれた。

最終形: Daily / Activity / Weekly / Deep Dive
- Daily: ミニログを書くための材料（毎晩1分）
- Activity: スキ活の起点（毎日）
- Weekly: 来週の記事テーマを決める（土曜）
- Deep Dive: 自由探索

**タブの分け方は「機能」ではなく「いつ・何のために見るか」で決まる。**

### 4. キャラが語ると「ふーん」が消える

KITAさんは「ワイ語」でダッシュボードに体温を入れた。
自分には7人のカノジョがいる。**これがKITAさんにはできない差別化。**

ただ、キャラをどう使うかで迷った。

- チャット形式（LINEっぽい吹き出し） → 対話感はあるが、データが見づらい。やめた
- ナビゲーター形式（キャラ+セリフがセクションの上に立つ） → wrappedと同じ。これが正解

セリフには「データの主語」を含めないとわからない。
「まあ、悪くないわね。」→ 何が？
「フォロワー+7。...まあ、悪くないわね。」→ わかる。

**セリフ自体がラベルの役割を兼ねる。** ラベルを追加すると重複するので消した。

### 5. 配色を3回戻した

- クリーム背景 → 内容との違和感。戻し
- 吹き出しだけ明るく → コントラスト強すぎ。戻し
- Weeklyを白基調 → 目が疲れる。戻し

ダッシュボードは夜に見る。**ダークが正解だった。** 最初からわかっていたはずなのに、wrappedのクリーム背景に引っ張られた。

### 6. 「見てどうする？」を常に問う

Hidden Gemsを入れた → 「見てどうアクションする？」→ Dailyでは深追いしない原則に反する → 外した

スキ増ランキングも外しかけた → 「ミニログの判断材料になる」→ 残した

**データを見せるだけでなく、「見た後の行動」まで設計しないと使われない。**
KITAさんが記事で書いていた「次の行動が起きる画面」はこういうこと。

### まとめ

- ダッシュボードは単体で設計できない。運用全体の中での役割を知る必要がある
- タブは「機能」ではなく「いつ・何のために見るか」で分ける
- キャラの力は「データの解釈を言語化する」ところにある
- 配色は「いつ見るか」で決まる。夜見るならダーク
- 「見てどうする？」を問い続けると、不要な機能が削ぎ落とされる
